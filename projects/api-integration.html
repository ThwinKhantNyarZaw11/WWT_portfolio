3rd Party API Integration

Overview

Built a multi-API dashboard application that fetches and displays real-time data from various public APIs, including weather, news, currency exchange, and GitHub activity. The goal was to demonstrate the ability to handle multiple asynchronous data sources with clean UI updates and error handling.

Key Features

  • Integrated multiple REST APIs (OpenWeather, NewsAPI, Fixer.io, GitHub)
  • Used Axios with async/await for cleaner request handling
  • Displayed live weather by geolocation and custom city input
  • Cached API responses for improved performance
  • Error boundaries and fallback states for failed API calls

Technology Stack

  • Frontend: React, Tailwind CSS
  • State Management: React Context + useReducer
  • APIs Used: OpenWeatherMap, NewsAPI.org, Fixer.io, GitHub REST
  • Security: Stored API keys securely using .env and backend proxying

Challenges & Solutions

  • Challenge: Coordinating multiple async requests on page load
  • Solution: Used Promise.all and conditional rendering strategies
  • Challenge: Avoiding API rate limits
  • Solution: Introduced debounce, cache layers, and retry logic

Impact

  • Demonstrated robust API integration handling for real-world scenarios
  • Inspired a modular architecture adopted in later client dashboards
  • Sharpened debugging, fetch logic, and UI resilience patterns
